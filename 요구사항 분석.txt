SImpleBoard

요구사항 분석

1. 해당 도메인에 대하여
 - 게시판과 게시글을 관리하는 프로젝트를 진행한다.
 - 게시판 분야는 여러 커뮤니티의 기본이 되는 구조이다.
 - 일반적인 게시글은 텍스트와 이미지 파일로 구성된다.  

2. 기능적 요구사항
 - 게시판
	-> 게시판은 생성될 수 있다.
		-> 생성 권한 필요
	-> 게시판은 게시판 명을 지니며, 수정될 수 있다.
		-> 수정 권한 필요
	-> 게시판은 삭제 될 수 있다.
		-> 삭제 권한 필요
	-> 게시판은 아래의 항목으로 구성된다.
		-> 게시판 제목
		-> 게시판 설명
		-> 게시판 생성 일자
		-> 게시판 수정 일자
	-> 게시판은 보유한 게시글을 페이징하여 보여준다.
	
 - 게시글
	-> 게시글은 생성 될 수 있다.
		-> 생성 권한 필요
	-> 게시글은 내용이 변경될 수 있다.
		-> 수정 권한 필요 && 작성자 본인
	-> 게시글은 삭제 될 수 있다.
		-> 삭제권한 필요 && 작성자 본인
	-> 게시글은 아래의 항목으로 구성된다.
		-> 게시글 제목
		-> 게시글 내용
		-> 게시글 작성자
		-> 게시글 첨부파일 [ 선택사항 ]
		-> 게시글 생성 일자
		-> 게시글 수정 일자
	-> 게시글은 댓글을 지닐 수 있다.
	-> 게시글은 보유한 댓글을 페이징하여 보여준다.

 - 첨부파일
	-> 각 첨부파일은 게시글 등록 때 같이 등록된다.
	-> 첨부파일은 특정 위치에 upload 되며 개별 식별자를 지닌다.
	-> 첨부파일의 식별자는 UUID와 fileName을 조합하여 사용한다.
	-> 첨부파일의 업로드 위치는 아래와 같다.
		-> c://upload
	-> 첨부파일은 이미지 파일만 허가한다.
	-> 첨부파일은 게시글이 등록 될 때 선택사항으로 같이 등록된다.
	-> 첨부파일은 게시글이 삭제될 때 같이 삭제된다.
	-> 첨부파일은 아래의 항목으로 구성된다
		-> 원본 파일 명
		-> UUID
		
 - 댓글
	-> 댓글은 생성 될 수 있다.
	-> 댓글은 수정 될 수 있다.
	-> 댓글은 삭제 될 수 있다.
	-> 댓글은 아래의 항목으로 구성된다.
		-> 댓글 내용
		-> 댓글 작성자 정보
		-> 댓글 작성 일자
		-> 댓글 수정 일자

3. 비기능적 요구사항
 - 회원들은 별도의 계정 생성을 하지 않고 OAuth2를 사용하여 로그인한다.
 - OAuth2를 지원하는 항목은 아래의 순서대로 적용한다.
	-> GOOGLE
	-> KAKAO
	-> NAVER
 - 세션 정보를 유지한다.
 - 회원 정보
	-> 회원 정보는 아래의 항목으로 구성된다.
		-> 회원 email
		-> 회원 password
		-> 회원 등급
		-> 회원 OAuth2 platform 정보
		-> 회원 허가 여부
		-> 회원 정지 여부
 - 회원 등급
	-> 회원 등급의 경우 해당 등급이 보유하는 권한 항목들을 지닌다.
	-> 등급 명명은 "등급명"으로 하고 코드에서 활용 시 ROLE_등급명 으로 변환하여 사용한다.
	-> 회원 등급은 아래의 항목으로 구성된다.
		-> 회원 등급 명
		-> 회원 등급 설명
	-> 등급은 해당 등급이 개별 권한을 보유할 수 있다.	
 - 회원 권한
	-> 회원 권한은 회원 별 수행할 수 있는 권한을 의미한다.
	-> 권한은 개별적으로 부여할 수 있다.
	-> 권한은 특정 등급에 속할 수 있다.
	-> 권한은 특정 회원에게 개별적으로 부여 될 수 있다.
	-> 권한은 아래의 항목을 지닌다.
		-> 권한 명
		-> 권한 설명
